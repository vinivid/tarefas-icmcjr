# Handbook

## Convenções de Nomeação

### Arquivos

Os arquivos seguem o padrão:

* kebab-case para nomes compostos;
* nomes em minúsculo para diretórios.

### Classes e Componentes

Utiliza-se PascalCase para componentes e modelos.

## Estilo de Código

* TypeScript como linguagem principal;
* respostas em JSON;
* separação entre controllers, models, routes e middlewares.

## Boas Práticas

* reutilização de código;
* separação das responsabilidades;
* uso de funções específicas para acesso ao banco de dados;
* hash das senhas utilizando bcrypt;
* autenticação utilizando JWT.

## Ferramentas Utilizadas

* Node.js
* TypeScript
* Express
* MongoDB
* Docker
* JWT
* bcrypt
* Vitest
